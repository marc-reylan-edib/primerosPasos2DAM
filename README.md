PROYECTO README.md

Proyecto de Desarrollo en la Nube de una aplicación de un sistema de colas para parques de atracciones con sistema FIFO.

** APP DE SISTEMA DE COLAS PARA PARQUE DE ATRACCIONES.¡

--En nuestra app funcionará de esta manera:

1- El usuario compra la entrada a traves de una web de dicha atraccion 2- El usuario entra a la cola de dicha atraccion 3- Inicia sesión a través de la app usando su entrada 4- La aplicación le mostrara su turno de cola, le indicará cuanto tiempo de espera debe esperar 5- Cuando le toque su turno le llmaran y podra entrar a la atraccion 6- El usuario sale de la atraccion y podra repetir o subirse a otra atraccion

Y asi funciona nuestra aplicación de first in first out

REQUISITOS FUNCIONALES:

Requisitos funcionales La aplicación deberá ser funcional La aplicación deberá gestionar la cola de visitantes La aplicación deberá añadir visitantes al final de la cola La aplicación deberá eliminar los visitantes atendidos de la cola La aplicación deberá respetar el orden de llegada (FIFO) Turno La aplicación deberá mostrar el número de turno actual La aplicación deberá notificar a los visitantes cuando sea su turno La aplicación deberá permitir registrar visitantes con un ticket o número de turno La aplicación deberá permitir consultar el número de personas en espera La aplicación deberá calcular el tiempo estimado de espera La aplicación deberá mostrar el tiempo estimado de espera reportes La aplicación deberá generar reportes de turnos atendidos historialCola La aplicación deberá almacenar el historial de la cola para análisis La aplicación deberá permitir al perfil administrativo gestionar la cola

usuarios Los usuarios podrán registrarse en la plataforma Los usuarios podrán iniciar sesión en la plataforma Los usuarios podrán actualizar su información personal Los usuarios podrán visualizar las atracciones disponibles

clientes & cola & notificaciones Los clientes podrán unirse a una cola de atracción Los clientes podrán abandonar una cola si lo desean Los clientes podrán consultar su posición actual en la cola Los clientes podrán recibir notificaciones cuando su turno se aproxime Los clientes podrán registrar una entrada o ticket digital vinculado a su perfil

empleados Los empleados podrán acceder con un perfil de tipo administrativo Los empleados podrán gestionar las colas de todas las atracciones Los empleados podrán verificar, pausar o reordenar colas en caso necesario Los empleados podrán eliminar clientes de la cola en casos excepcionales (por incumplimiento o error) Los empleados podrán generar reportes de flujo de visitantes y rendimiento de las atracciones

entradas La aplicación deberá permitir registrar entradas asociadas a un cliente La aplicación deberá validar la autenticidad de una entrada antes de permitir el acceso a la cola La aplicación deberá registrar la hora y fecha de uso de la entrada

atracción La aplicación deberá mostrar información básica de cada atracción (nombre, capacidad, tiempo promedio de atención, estado) La aplicación deberá permitir a los empleados asignar colas a cada atracción. La aplicación deberá permitir modificar el estado de una atracción (activa, en mantenimiento, cerrada) estadística La aplicación deberá generar estadísticas de uso por atracción (número de visitantes, tiempos promedio, etc.)

REQUISITOS NO FUNCIONALES

Requisitos no funcionales La aplicación debe ser estética y amigable. La aplicación debe actualizar la cola en tiempo real La aplicación debe tener un tiempo de respuesta inferior La aplicación debe ser confiable y no perder datos de la cola por fallos La aplicación debe ser segura y proteger los datos de los visitantes La aplicación debe ser compatible para dispositivos móviles y ordenadores La aplicación debe manejar un gran número de visitantes La aplicación debe permitir que muchos administradores puedan gestionar la cola.
